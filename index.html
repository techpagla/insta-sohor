How does JavaScript works?

JavaScript is a programming language that runs on the client side (in the browser) rather than the server side. When a user requests a web page that contains JavaScript, the script is sent to the user's browser along with the HTML and CSS of the page. The browser then interprets and runs the JavaScript code, allowing it to update and manipulate the content of the page, handle user interactions, and make requests to the server for new data. This allows for dynamic and interactive websites.

what does javaScript even loop do?

An event loop in JavaScript is a mechanism that allows the execution of code to be scheduled and coordinated. It is responsible for handling the execution of JavaScript code in a non-blocking, asynchronous way. JavaScript is single-threaded, which means that it can only process one task at a time. The event loop is the mechanism that allows JavaScript to handle multiple tasks at the same time, by managing the execution of code in a way that doesn't block other tasks from being executed. The event loop works by continuously monitoring the message queue, which is a queue of messages or tasks that need to be executed. Each message represents a unit of work, such as a function call or an HTTP request. The event loop takes the messages from the queue one by one, and executes them in order. If a message takes a long time to execute, it is put into the "backend" and the event loop continues to execute the next message, this way the event loop doesn't block the whole execution and the application keeps responsive. The event loop is an important aspect of JavaScript and is used in many JavaScript APIs and frameworks, such as Node.js and web browsers.

Local vs Session storage

Local storage and session storage are both types of web storage that allow web applications to store data in the user's browser. However, they differ in the length of time that the data is stored and the scope of the data. Local storage stores data indefinitely until it is explicitly cleared by the user or the web application. Session storage stores data only for the duration of the user's browser session, which means that the data is lost when the user closes the browser or navigates away from the website. In short, Local storage is permanent and session storage is temporary and will be deleted when the browser is closed.
